RDF 與書目資料

RDF 是資源描述框架，用三元組描述資源與資料關係。對圖書館編目而言，它讓書目資料可以從單筆紀錄轉化為可連結、可查證、可重複使用的知識結構。

本單元目標

  • 理解 RDF 是一種以三元組描述資源之間關係的資料模型，而不是單純的資料格式。
  • 區分書目紀錄中的資料單位與語意單位。
  • 說明 RDF 如何讓傳統編目資料轉化為可連結、可重複使用、可與其他資料集整合的 linked data。

RDF 與書目紀錄

RDF 的中文標準名稱為資源描述框架。它是一種用來描述網路資源與資料關係的資料模型。

RDF 的基本想法，是將資料拆解為一組組主詞、述詞、受詞的三元組，藉此表達某個資源與另一個資源，或某個資源與特定值之間的關係。對圖書館編目而言，RDF 的重要性在於：它讓書目資料不只是被放在一筆紀錄中，而是可以被拆解為作品、作者、主題、版本與館藏單件等實體，並透過明確的語意關係彼此連結。

編目過往通常以單筆書目紀錄作為主要單位。一筆 MARC 紀錄可能同時包含題名、作者、出版者、出版年、主題、館藏資訊等內容。這種做法很適合館藏管理與目錄查詢，但當資料要放到網路環境中，並與其他機構、資料庫或知識系統連結時，就會遇到限制。

如果使用 RDF 模型來重組書目紀錄，重點就不再只是這筆紀錄有哪些欄位，而是這筆紀錄在描述哪些實體，以及這些實體之間有什麼關係。例如，《紅樓夢》是一個作品，曹雪芹是一位作者，清代小說是一個主題概念，某個出版社出版的版本則是一個具體版本。在 RDF 模型中，這些對象會被拆成不同的實體，再用關係把它們連起來。

也就是說，編目資料不只是紀錄中的欄位，也可以被理解為一組實體與實體之間的語意關係。

從單筆書目紀錄到多組實體關係

一筆傳統書目紀錄通常會把許多描述資訊集中在一起。例如，一筆描述《紅樓夢》的 MARC 紀錄，可能同時包含題名、作者、版本、出版者、出版年、主題詞、分類號與館藏資訊。從目錄查詢的角度來看，這些資訊放在同一筆紀錄中很方便，但從 RDF 的角度來看，這些資訊其實可以拆成多個不同的實體與關係。

拆分的第一步，是先找出紀錄中提到哪些對象。以《紅樓夢》為例，作品本身可以是一個實體，曹雪芹可以是一個作者實體，清代小說可以是一個主題概念，某個出版社於某年出版的版本可以是一個版本或實例，圖書館實際收藏的那一本書則可以是一個館藏單件。這些對象在傳統紀錄中可能只是分散在不同欄位中的文字，但在 RDF 模型中，可以被視為彼此獨立、可被識別的實體。

第二步，是把這些實體之間的關係說清楚。例如，《紅樓夢》這個作品有作者曹雪芹，也有主題清代小說，某個出版版本是《紅樓夢》的一個具體版本，某個館藏單件則是圖書館持有的實體物件。用 RDF 的思維來看，這些都可以改寫成一組組關係：作品連到作者、作品連到主題、版本連到作品、館藏單件連到版本。如此一來，書目資料就不只是欄位的集合，而是一個由實體與關係構成的網絡。

《紅樓夢》 → 作者 → 曹雪芹
《紅樓夢》 → 主題 → 清代小說
出版版本 → 版本屬於 → 《紅樓夢》

這種拆分方式的好處，是資料可以被重複使用與跨系統連結。曹雪芹不必在每一筆書目紀錄中都被當成文字重新輸入，而可以成為一個穩定的作者實體；不同版本的《紅樓夢》都可以連回同一個作品實體；不同圖書館的館藏單件也可以連到各自收藏的版本。當資料被整理成這種形式時，系統不只知道某筆紀錄中出現了哪些文字，也能理解這些文字背後指向哪些對象，以及它們彼此之間有什麼關係。

Linked Data（鏈結資料）

Linked data 是一種在網路環境中發布與連結資料的做法。它鼓勵資料提供者使用穩定 URI 識別資源，並用 RDF 描述資源之間的關係。如此一來，不同機構建立的作者、作品、主題或館藏資料，就能彼此連結，形成可被機器讀取與重複使用的資料網絡。

RDF 的核心：三元組與圖

RDF 最基本的表達單位是三元組，也就是 subject、predicate、object。它可以理解為一句最小的資料敘述：某個對象，透過某種關係，連到另一個對象或某個值。當許多三元組彼此連接時，就會形成 RDF graph，也就是由節點與關係構成的資料圖。

S

Subject

Subject 是這句話主要在描述的對象，也就是我們要談的是誰或什麼。例如，《紅樓夢》— 作者 — 曹雪芹這個三元組中，《紅樓夢》是 subject。

P

Predicate

Predicate 是 subject 和 object 之間的關係，通常可以理解為實體間的某種關係或實體擁有的屬性。例如，作者是 predicate，表示作品和作者之間的關係。

O

Object

Object 是這個關係所連到的另一個對象，或是某個具體的值。例如，曹雪芹是 object；若換成出版年，object 也可以是 1980 這樣的年份值。

三元組與 node、arc、node

RDF 三元組也可以畫成一個簡單的圖形結構：node、arc、node。Subject 和 object 通常會成為圖中的 node，predicate 則是連接兩個 node 的 arc。換句話說，三元組不是只適合用句子理解，也可以被視為圖中的一條有方向的關係。

《紅樓夢》 → 作者 → 曹雪芹
Work A → hasSubject → Topic B
Instance A → instanceOf → Work B

Node

Node 是 RDF graph 中的節點，可以代表作品、作者、主題、機構等實體，也可以代表文字、日期或數字等值。在三元組中，subject 和 object 通常會成為 node。例如，《紅樓夢》和曹雪芹都可以是 node。

Arc

Arc 是連接兩個 node 的有向線段，用來表示兩者之間的關係。在三元組中，predicate 通常對應到 arc。例如，《紅樓夢》— 作者 — 曹雪芹中，作者就是從《紅樓夢》指向曹雪芹的 arc。

RDF graph：由多個三元組組成的資料圖

一個三元組只能描述一個最小關係，但書目資料通常包含許多關係。因此，RDF 會把多個三元組連接起來，形成一個 RDF graph。以《紅樓夢》為例，作品可以連到作者曹雪芹，也可以連到主題清代小說，某個出版版本可以連到作品，某個館藏單件又可以連到出版版本。這些三元組彼此連接後，就形成一個以作品、作者、主題、版本與館藏單件為節點的資料圖。

這對編目很重要，因為傳統欄位中的內容可以被重新理解為一組組三元組。題名、作者、主題、出版年、語言等，不只是欄位值，而是可以被表達成某個資源具有某種屬性，或某個資源與另一個實體有某種關係。因此，RDF 提供了一種把書目資料轉化為語意網絡的方法。

另外，RDF 很重視識別。若只是寫曹雪芹，系統不一定知道它指的是哪一個人；若使用穩定 URI，則可以讓不同資料集指向同一個對象。這也是 RDF 與 linked data 能夠支援跨系統連結的基礎。

RDF、RDFS 與 OWL 的角色

RDF 是用三元組描述資源與關係的資料模型。它可以讓我們寫出像 Work A — hasAuthor — Agent B 這樣的敘述，表示某部作品與某位作者之間的關係。不過，當資料規模變大時，只寫出一組組三元組還不夠，我們還需要說明這些三元組中使用的詞彙代表什麼。

例如，資料中可能會出現 Work、Agent、hasAuthor、hasSubject 等詞彙。這時就需要進一步說明：Work 和 Agent 是什麼類型的概念？hasAuthor 這個關係通常連接哪兩種對象？某個資源為什麼可以被判定為作品或作者？RDFS 與 OWL 的角色，就是在 RDF 的基礎上，提供這些類別、屬性與關係的語意說明。

RDF

負責表達個別資料敘述，例如某部作品有作者、某個版本屬於某部作品。

RDFS

提供基本的類別與屬性結構，說明資料中使用的類別、屬性、domain 與 range。

OWL

提供更完整、更精細的語意模型能力，能描述屬性類型、類別關係與推論限制。

RDFS：提供基本的類別與屬性結構

RDFS 的全稱是 RDF Schema。它是建立在 RDF 之上的基本語意工具，主要用來說明資料中使用的類別與屬性。換句話說，RDFS 可以幫助我們描述資料模型中的基本結構，例如哪些資源屬於哪一類、某個屬性通常適用於哪一類資源，以及它的值通常應該是哪一類對象。例如：

ex:Van → rdfs:subClassOf → ex:MotorVehicle

rdfs:subClassOf 用來定義類別之間的階層關係。如果 A 類別是 B 類別的子類別，那麼所有屬於 A 的實例，也同時會是 B 的實例。卡片中的意思是：Van 是一種 MotorVehicle。

hasAuthor → rdfs:domain → Work

rdfs:domain 用來指出某個 property 被使用時，三元組中的 subject 應該屬於哪一個類別。它可以視為一種限制，表示凡是使用這個 property 的資源，通常應該是指定類別的實例。卡片中的意思是：只要某個資源使用了 hasAuthor 這個屬性，那這個資源就應該被視為 Work 類別的東西。

ex:hasAuthor → rdfs:range → ex:Person

rdfs:range 用來指出某個 property 被使用時，三元組中的 object 應該屬於哪一個類別，或應該是哪一種資料型別。它可以視為一種限制，表示這個 property 可以連接到哪些類型的值。卡片中的意思是：只要某個資源使用了 hasAuthor 這個屬性，那麼這個屬性所指向的對象，就應該被視為 Person 類別的東西。

這表示 hasAuthor 這個關係通常是從 Work 連到 Agent。換句話說，當我們看到某個資源透過 hasAuthor 連到另一個對象時，可以理解前者通常是一個作品，後者通常是一個作者或創作者類型的實體。因此，RDFS 的重點在於提供基本的語意框架，讓 RDF 資料中的類別與屬性不只是名稱，而是具有可被理解的結構位置。

OWL：提供更完整的語意模型能力

OWL 的全稱是 Web Ontology Language。它同樣建立在 RDF 的基礎上，但比 RDFS 提供更完整的語意模型能力。若說 RDFS 主要用來描述基本的類別與屬性，那麼 OWL 則可以更細緻地描述類別之間、實體之間，以及屬性之間的關係。

Ontology（本體）

Ontology（本體）可以理解為某個知識領域中的概念模型。它會說明這個領域有哪些重要類別、這些類別有哪些屬性，以及不同類別之間有什麼關係。在圖書館資料中，ontology 可以用來描述作品、作者、版本、主題、館藏單件等概念，以及它們彼此之間的關聯。

在 OWL 中，被描述的具體對象可以稱為 individuals，概念類型稱為 classes，關係或屬性稱為 properties。例如，Jane Austen 可以是一個 individual，Agent 可以是一個 class，而 hasAuthor 則可以是一個 property。

Pride and Prejudice → hasAuthor → Jane Austen
Pride and Prejudice → publicationYear → 1813

相較於 RDFS，OWL 的重點在於能表達更細緻的語意限制與推論關係。對圖書館資料來說，這表示作品與版本、作者與機構、作品與主題概念之間的關係，可以用更精細的語意模型來描述，使書目資料不只是可查詢的紀錄，也能成為可推論、可整合的知識結構。

從描述資料到知識圖譜

RDF 與編目的關係，可以理解為把編目資料從紀錄格式轉化為語意連結。傳統編目強調如何正確描述一件資源，例如題名怎麼著錄、作者怎麼標目、版本資訊怎麼記錄。RDF 並不是要取代這些編目知識，而是提供另一種資料表達方式，讓這些描述可以被機器理解、連結與再利用。

以 BIBFRAME 為例，它試圖把書目資料轉為較適合 linked data 環境的模型。作品、實例、單件、作者、主題等不再只是欄位中的文字，而可以成為可識別的實體。這使圖書館資料更容易和其他資料來源連結，例如權威資料、主題詞表、出版資料、研究資料庫或文化資產資料庫。

從這個角度來看，RDF 可以作為整理書目資料、建構知識圖譜的基礎。知識圖譜的重點，是把不同對象拆成節點，並用明確的關係把它們連接起來。RDF 的角色，則是用三元組把這些節點與關係表達出來。當作品、作者、主題、版本與館藏單件都能成為節點，並透過作者、主題、版本、館藏等關係彼此連接時，書目資料就不只是用來顯示單筆紀錄，而能形成一個可查詢、可連結、可推論的知識網絡。

練習題目

  1. 請將下列敘述改寫成 RDF 三元組形式：《傲慢與偏見》的作者是 Jane Austen。
  2. 請判斷下列哪些比較適合視為可識別的實體，哪些比較像文字、年份或語言等值：Jane Austen、1813、English、Pride and Prejudice、臺北市立圖書館。
  3. 請用 RDF 的思維，拆解一筆書目紀錄中可能包含的實體。至少列出：作品、作者、主題、版本或實例、館藏單件。
  4. 假設有一個 property 叫做 hasAuthor，請說明它的 subject 和 object 在圖書館書目資料中可能分別是什麼。
