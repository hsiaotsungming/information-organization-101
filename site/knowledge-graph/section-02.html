為什麼生成式AI下還需要知識圖譜？

大型語言模型讓我們更容易用自然語言接近知識，但資訊組織需要提供與維護可被辨識、查證的知識。

本單元目標

  • 分辨生成式AI與知識圖譜的知識處理方式。
  • 理解大型語言模型產生答案時的限制。
  • 說明知識圖譜如何補足大型語言模型的限制。

生成式AI回答的本質

既然大型語言模型已經可以回答問題、摘要、翻譯、改寫，為什麼還需要知識圖譜？

生成式AI的回答，本質上像是在接續一段文字。它會根據使用者的問題、前後文，以及模型在大量文本中學到的語言模式，推估接下來最可能出現、最符合語境的字詞，逐步組合成一段看起來通順、貼近問題的回答。但是，產生回答和組織知識是不同的工作。

生成式AI的限制

大型語言模型擅長語言生成與語意推測，不等於它本身就是可靠、可維護、可追溯的知識庫。雖然它能根據語境，來生成可能合理的回答，但回答中的概念、來源與關係，並不一定能是準確的。

例如，當我們問某位作家有哪些作品時，模型可能列出一串看似合理的書名。但如果其中牽涉筆名、譯名、不同版本、改編作品，模型就可能把不同層次的資料混在一起。以圖書館資料來說，「作品」、「譯本」、「版本」、「改編作品」其實不是同一件事，但生成式AI在回答時，不一定會嚴格區分這些差異。

另一個問題是來源與推論路徑不夠清楚。模型可能回答「這本書和女性成長、婚姻制度有關」，但不一定能說明這個判斷來自哪個主題詞、哪段摘要、哪筆書目資料。它很會組織語句，但不一定能讓使用者追溯「這個答案從哪裡來」。

這不是說模型沒有用，而是表示它的回答本質上是一段即時生成的文字，不會是穩定而可檢查、更新的紀錄。對圖書館或知識組織工作而言，這會造成維護上的困難。生成式AI可以產生答案，但答案是否有清楚來源、是否能被驗證、是否能被長期維護，需要依靠其它技術來協助。

知識圖譜的強項

知識圖譜補上的不是生成能力，而是結構能力。生成式AI可以根據語境產生流暢回答，但回答中的實體、來源與關係不一定清楚，也不一定容易驗證。知識圖譜則將知識整理成可以被系統辨識、連結、查詢與更新的形式，讓資料有明確身分之外，也讓資料之間的關係被清楚記錄。因此，資料不再只是分散的文字片段，而是形成可以檢查與維護的知識網絡。例如：

實體

實體讓系統能辨識知識中的對象。例如 Jane Austen 是一位作者，Pride and Prejudice 是一部作品，marriage 是一個主題。實體可以讓系統知道，這些字串各自代表的知識單位。

關係

關係讓系統能表示這些實體之間如何相連。例如 Jane Austen 寫了 Pride and Prejudice，而 Pride and Prejudice 涉及 marriage 這個主題。透過關係，系統可以知道知識的脈絡。

屬性

屬性則補充每個實體本身的描述資訊。例如 Jane Austen 有出生年代與國籍，Pride and Prejudice 有出版年與語言。

以《The Da Vinci Code》為例，這個著作有多種不同的資料，像是作品、翻譯版本、實體書、作者、主題、畫作與畫家。這些資料各自有明確身分，也透過「由誰撰寫」、「關於什麼」、「由誰繪製」、「哪個圖書館持有」等關係連在一起。下面是一些由圖中可以辨識出來的實體、關係、屬性。而以知識圖譜呈現時，書目資料可以被系統清楚辨識、查詢與驗證。

實體

The Da Vinci Code 是一個作品層級 Work 的實體。

實體

這個著作有 Dutch translation 表現形式 Expression 的實體。

屬性

The Da Vinci Code 這個 Work 的 URI 是 http://openlibrary.org/works/OL76837W。

關係

The Da Vinci Code 由 Dan Brown 撰寫 written by。

關係

圖書館持有 Holding 某一本書的館藏單件 Item。

所以，當生成式AI參考知識圖譜來回答問題時，可以依據已整理好的實體與關係生成答案。這有助於降低名稱混淆、關係誤判與細節編造的風險，也能補強生成式AI在可驗證性、可追溯性與知識維護上的不足。

兩種方法的強弱項目

生成式AI

強在語言理解與文字生成，但較難保證答案背後的資料結構、來源與維護狀態。

自然語言與脈絡說明

能理解使用者用自然語言提出的問題，改寫需求、補足語境，並把結果組織成可讀的說明。

產生答案

能快速生成連貫、貼近情境的文字，但答案中的細節不一定都有明確資料來源。

資料更新

回答可能受訓練資料、提示方式與當下語境影響，不適合單獨作為長期維護的知識紀錄。

知識圖譜

強在結構化、可辨識與可追溯，能把答案連回穩定的實體、關係與資料來源。

實體與關係對應

能把自然語言需求對應到明確的實體與關係，並提供關係路徑作為解釋基礎。

消歧義與查證

能區分相似名稱、不同版本與不同資料層級，並提供可查證的資料節點與連結。

資料維護

能支援資料更新、版本管理與來源追溯，讓知識可以被長期檢查與維護。

資訊組織的角色

從資訊組織的角度來看，除了要保存資料之外，更要讓資料可以被發現、辨識、連結成脈絡，由此提供資訊。使用者從關鍵字檢索轉向自然語言或生成式檢索，這不代表編目、權威控制、主題分析與資料模型不再重要，而是它們更可能退到回答背後，成為支撐 AI 回應品質的知識基礎。

圖書館實務上，BIBFRAME、IFLA LRM、VIAF，以及臺灣國家圖書館與臺大合作的 LDT@Library，都可視為把敘述編目與主題編目的內容，轉成可連結知識圖譜的概念。

BIBFRAME

BIBFRAME 是由美國 Library of Congress 推動的書目資料模型，目的在於讓圖書館書目資料更適合 linked data 環境。它將書目描述整理為 Work、Instance、Item 等核心層次，並以 RDF vocabulary 表示實體、屬性與關係，使書目資料能被識別為可連結的 Web resources。

IFLA LRM

是整合 FRBR、FRAD 與 FRSAD 而形成的書目概念模型，用來統一描述書目資料中的資源、行動者與主題關係。它以 entity-relationship 架構呈現 bibliographic universe，並將 Work、Expression、Manifestation、Item 與 agent、subject、place、time-span 等實體納入同一模型。

LDT @ Library

臺灣圖書館界推動 linked data 應用的合作計畫，背景在於國內部分圖書館自 2018 年起陸續導入支援 linked data 的系統，並開始轉向 RDA 編目規範。此計畫由國家圖書館與國立臺灣大學圖書館合作，整合權威、題名與主題資料，建置「臺灣鏈結資源系統」。

資訊組織的專業不只是整理資料，也是在整理資料的過程中建構知識。在這裡，知識建構代表：判斷哪些事物應被視為實體、哪些特徵需要被記錄、哪些關係必須被清楚標示。透過這樣的分析與組織，資料得以形成描述知識的結構，使 AI 在回應使用者問題時，更有機會依據穩定的知識結構來組織答案，而不是只依賴語言上的相似性進行推測。

因此，圖書館的編目規範、權威控制與主題分析，可視為一種分析與建構知識環境的能力。當資料進一步轉化為 linked data 或知識圖譜，圖書資訊專業者就能在資料、系統與使用者之間，設計可以理解、連結與驗證的知識結構，以此強化 AI 生成回答時所依據的知識基礎。

檢索上的結合應用

生成式AI可以先判斷問題中的重要條件，例如人物、作品主題、語言與版本需求，再依這些條件查找知識圖譜。知識圖譜提供可辨識、可驗證的實體與關係，讓系統找到適當內容後回傳給生成式AI後，再由生成式AI把這些結構化結果整理成容易閱讀的回答。

問題例：我想找 Jane Austen 作品中和婚姻主題有關的中文譯本。

在這樣的合作檢索中，大語言模型可以分析檢索句中的檢索重點，例如使用者要找的是特定作者、特定主題、中文譯本，以及不同版本或館藏線索。知識圖譜的話，則是提供可查證的資料，例如 Jane Austen、相關作品、marriage 主題、中文譯本與館藏線索。生成式AI再把查得的內容，組織成使用者容易理解的回答。因此，回答不只是依語言機率性生成的文字，而是可以連回具體的實體、關係與資料來源。

小練習

  1. 說明為什麼生成式 AI 可以從概念上理解為根據上下文預測下一個詞的機率模型。
  2. 選一個中文查詢問句，指出其中哪些詞可能是實體，並說明這些實體之間可能有什麼關係。
  3. 說明單純使用生成式AI進行問答時，可能會遇到哪些限制或風險。
  4. 描述知識圖譜可以如何協助生成式AI，讓回答更容易被查證與維護。
