05 Knowledge Graph

知識圖譜

知識圖譜將資料中的實體、概念與關係轉化為可計算的語意網絡。它不只描述「有哪些資料」，也描述資料之間如何互相指涉、支撐與推論。

學習目標

  • 辨識知識圖譜中的實體、屬性與關係。
  • 理解三元組如何表達一筆可被機器處理的知識陳述。
  • 說明知識圖譜與編目、分類、權威控制之間的關聯。
  • 以圖譜觀點重新組織一組課程或館藏資料。

核心概念

知識圖譜的重點不是把資料畫成圖，而是讓資料中的關係具備明確語意。 當關係被清楚命名，資料就能被查詢、連結、比較與推論。

Entity

實體

可被識別的對象，例如人物、作品、主題、機構、地點或事件。

Relation

關係

實體之間的語意連結，例如作者、改編自、屬於、位於或研究主題。

Attribute

屬性

描述實體的資料值，例如名稱、日期、識別碼、語言或版本。

建置流程

1. 界定範圍

決定圖譜要回答的問題與資料邊界。

2. 建立模型

定義實體類型、關係類型與必要屬性。

3. 轉換資料

將表格、書目或文字資料整理為一致格式。

4. 查詢應用

透過檢索、視覺化或推論呈現資料關係。

三元組範例

主詞 Subject 謂詞 Predicate 受詞 Object
西遊記 作者 吳承恩
西遊記 主題 中國古典小說
吳承恩 身分 作家

三元組讓每一筆知識陳述都具備固定結構，也讓不同資料來源之間能夠進行語意對接。

與編目的關係

傳統編目重視資源的描述與檢索點，知識圖譜則進一步把檢索點轉化為可連結的實體。作者、作品、主題標目、分類號與機構名稱，都可以成為圖譜中的節點。

從這個角度看，權威控制不只是名稱一致化，也是在建立穩定的實體識別；主題編目不只是選擇標題詞，也是在安排知識網絡中的語意位置。

課堂討論

  1. 如果要為一門課建立知識圖譜，哪些內容適合成為實體？
  2. 同一個人物在不同資料來源有不同名稱時，圖譜應該如何處理？
  3. 知識圖譜能解決哪些傳統關鍵字檢索不容易處理的問題？
